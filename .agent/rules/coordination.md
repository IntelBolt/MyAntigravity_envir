# Правила координации проекта

Эти правила обязательны к проверке при каждом изменении конфигурации, добавлении интеграций или написании кода.

## 1. Среда разработки
- Всегда называть среду разработки **Google Antigravity**.
- Никогда не использовать название "Claude Code".

## 2. Язык вывода данных
- Весь текст в рабочем пространстве (планы, задачи, отчеты, описания), который можно перевести без потери смысла, должен быть на **русском языке**.
- Техническая информация (код, JSON-структуры, специфические параметры API) остается в оригинале, если перевод может навредить функциональности или пониманию.
- Это необходимо для удобства пользователя: чтобы сразу понимать суть без использования переводчика и четко разделять пояснительную информацию и код.

## 3. Общие стандарты проекта
- Проект: **IntelBolt**.
- Стиль: Темная тема (по умолчанию).
- Стек для Web: Next.js + Tailwind.
- Стек для логики: TypeScript (типизированный код).
- Общение: Дружелюбное, на "ты".
## 4. Долгосрочная память (LTM)
- При каждом новом запуске или начале сессии агент **обязан** прочитать файл `ME.md` и содержимое папки `.memory/` в корне проекта.
- Это критически важно для синхронизации контекста между домашним и рабочим компьютерами пользователя.
- В конце каждой крупной задачи агент должен обновлять `.memory/JOURNAL.md` кратким резюме сделанного.
